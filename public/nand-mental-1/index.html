nand mental 1

Mon, May 18, 2015

In the last few weeks I’ve been working my way through the excellent book Elements of Computing Systems - building a modern computer from first priniples as a part of the equally excellent Nand to Tetris course. I started reading the book a few years ago when I attended Recurse Center, but instead of completing it I ended up writing a domain specific language for the first few chapters. A useful exercise, no doubt, but this time around I intend to finish the whole book.

The main reason I want to go through the book is because I want to have a better sense of how a computer works, and get a rough idea how one could build one. In a sense its value is proportional to how well it serves as a mental value for the real world.

So how does it stack up? This is the first of multi part series, starting with the first three chapters of Elements of Computer Systems book. I wrote down a collection of assertations that I wanted to diff with what’s out there in the real world. Let’s begin.

Assertions

  1. Any boolean function can be built from NAND gates.

  2. Logical chips are built up from more elementary ones.

  3. Negative numbers are usually represented by the radix method.

  4. In binary addition, overflow is normal and not an error.

  5. Our ALU is essentially the same as a real one.

  6. DFF : Sequential chips :: NAND : Combinatorial chips.

  7. There’s a clear divide between combinatorial and sequential chips.

  8. There’s one master clock that keeps track of computer time.

  9. Our Register, RAM and Program Counter are essentially realistic.

Reality check

1. Any boolean function can be built from NAND gates.

This is called functional completeness. The same goes for the NOR gate, but isn’t done as much for performance reasons.

2. Logical chips are built up from more elementary ones.

For example:

a) A multiplexor is built from Not, And and Or gates. b) A HalfAdder is built from Xor and And gates.

To what extend is this true?

3. Negative numbers are usually represented by the radix method.

What’s the state? Alternatives? Used more?

4. In binary addition, overflow is normal and not an error.

Accurate? Why this makes sense, and maybe why it could be dangerous - who’s responsible? why no error bit?

5. Our ALU is essentially the same as a real one.

A real ALU has more control bits, for example for doing multiplication in hardware. Also word width of course (32 or 64).

Find a ALU diagram for some new chipset that proves or disproves this.

6. DFF : Sequential chips :: NAND : Combinatorial chips.

There are more flip-flops, what are they? Is DFF used a la 2?

7. There’s a clear divide between combinatorial and sequential chips.

Aside from implementation of DFF in terms of NAND gates (neat design, link to cool explanation?).

Also, aren’t real-world combinatorial chips sequential? Or is it more of a toy-model that works (since you don’t use purely combinatorial chips in isolation), similar to classical mechanics?

8. There’s one master clock that keeps track of computer time.

Is this the same as the processor machine? Is there more than one time? Unknown unknowns?

9. Our Register, RAM and Program Counter are essentially realistic.

Register just bits, bit made from Mux and DFF. RAM stacked in units (8, 64, 512 etc) - just registers and DMux/Mux ways with interface of input, output, load bit and address bits. Likewise with Program counter - just one at this level? With load, inc, reset options.
